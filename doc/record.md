# MaCA 多智能体空战 RL 项目阶段记录（4.19）

## 一、当前项目阶段
当前主线已从早期 DQN 思路彻底转向 **recurrent MAPPO + centralized critic**。  
在关闭若干高耦合控制面后，已经得到一个**能稳定第一波接敌并完成部分击杀**的可训练基线：

- 关闭高低层 mode 决策
- 关闭雷达搜索 RL 控制
- 关闭复杂导弹发射 RL 控制
- 当前主要验证环境：`fix_rule_no_att`

这说明：**主训练链路已经跑通，但策略仍停留在“会打一波”的阶段。**

---

## 二、今天确认的核心问题
当前策略的主要缺陷不是“不会战斗”，而是**不会主动搜索与残局重搜索**：

1. **整体运动偏置明显**  
   己方飞机倾向于朝右上角推进并最终收缩，形成单角聚集。

2. **第一波能打，但残局不会补扫**  
   第一波接敌时可以造成一定击杀，但若有漏网之鱼，队伍不会重新展开并二次接敌。

3. **团队协同没有真正形成**  
   某架飞机即使已经没弹，但仍能持续跟踪敌人；然而其他飞机不会据此产生明显支援/接管动作。

4. **原有搜索 shaping 过弱**  
   先前加入的 `search_reward` / `reacquire_reward` 在日志中几乎为 0，说明这类轻量 reward 没有真正推动策略学习。

---

## 三、今天的重要判断
### 1. 纯 DQN 方案不适合作为最终主路线
主分支保留的 DQN 更像早期教学/验证用方案。对于当前这种：

- 多智能体协同
- 部分可观测
- 长时序搜索—战斗—重搜索
- 奖励稀疏且带明显 team credit assignment 问题

的场景，**纯 DQN 端到端很难稳定学出有效策略**。  
它可能学出局部反应式追击，但不太可能自然学出成熟的协同搜索与残局围捕。

### 2. 问题本质已从“战斗控制”转向“多机协同搜索”
当前项目的关键矛盾已经不是单机控制，而是：

- 如何主动搜索全场
- 如何利用团队敌情进行目标接力
- 如何避免全队扎堆单区域
- 如何完成漏敌后的全场补扫与收尾

也就是说，**多智能体协同的意义开始真正显现出来**。

---

## 四、今天放弃/修正的错误方向
### 错误方向：固定 `search_target_x` 前推
先前的 fallback 搜索思路，本质上仍是：
- 向敌方半场某个固定 x 推进
- 再加弱展开项

这个思路只对**开局**有一点意义。  
在第一波交战后，敌人可能已经进入我方半场甚至遍布全场，因此继续沿用固定前推目标，会把策略推向错误的整体偏置。

### 修正结论
后续搜索策略不能再基于“固定前推点”，而应该基于：

> **全场区域优先级评估（priority map）**

---

## 五、今天确定的新方案（阶段 B）
本次代码改造的目标是：

> **用 coarse priority map 替代固定前推搜索，让 AI 学会“主动找人”。**

### 核心设计
1. **环境侧构建 coarse priority map teacher**
   - 粗网格划分全场（如 4x4 / 5x5）
   - 每个区域综合评估：
     - 当前敌情证据
     - 未覆盖时长 / 不确定性
     - 最近敌情扩散残余
     - 友机拥挤与已分配惩罚
   - 有敌情时更像 threat map，无敌情时更像 search-value map

2. **模型侧新增 priority CNN 辅助头**
   - 基于 semantic screen CNN 特征，预测 coarse priority map
   - 通过 teacher map 做辅助监督

3. **actor 真正使用 top-K 区域信息**
   - 从 priority map 取 top-K 区域
   - 将其转成低维结构特征（相对位置、score、不确定性、entropy）拼入 actor 输入
   - 让策略决策实际依赖区域优先级，而不只是“看图 + 自己反应”

4. **目标从“追点”升级为“扫区域”**
   - 无敌情时：优先搜索长时间未覆盖区域
   - 有敌情时：围绕最近敌情残余区域进行补扫和接力
   - 通过区域分配/拥挤惩罚抑制全队扎堆

## 七、今天这一步的项目价值
今天最大的收获，不是“又加了一个 reward”，而是项目视角发生了升级：

> 项目问题已经从“单机体怎么追敌”升级为“机群如何协同完成搜索、接力、围捕与收尾”。

这对后续复盘和简历都很重要。因为它说明这个项目的技术重点已经不再是简单的 RL 套环境，而是：

- 多智能体协同搜索
- 团队敌情共享与任务接管
- 区域优先级建模
- 结构化工程启发与学习策略融合

## 八、可直接用于简历/面试的表述草稿
围绕 MaCA 多智能体空战环境，搭建 recurrent MAPPO 训练主线，并针对机群“第一波能接敌但残局不会补扫”的问题，逐步将搜索策略从固定前推 shaping 升级为 **coarse priority map 驱动的区域搜索与重搜索框架**；在环境侧构建区域优先级 teacher，在模型侧引入 semantic screen priority head 与 top-K 区域特征融合，推动策略从“碰见再追”向“主动找人、分区搜索、团队接力”演化。
